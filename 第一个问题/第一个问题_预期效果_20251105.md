# USDT与USDC综合对比分析系统 - 预期效果说明

## 一、程序概述

本程序是一个基于层次分析法(AHP)和模糊综合评价理论的稳定币竞争力评估系统。通过多维度指标体系，对USDT和USDC两种主流稳定币进行定量化对比分析，并生成详细的评价报告和可视化图表。

**开发语言**：Python 3.8+

**核心算法**：
- 层次分析法(AHP) - 用于权重计算
- 模糊综合评价 - 用于多维度综合评分
- 风险矩阵模型 - 用于风险评估

**主要功能**：
- 自动计算指标权重并进行一致性检验
- 多维度综合评分与对比分析
- 风险评估与等级划分
- 自动生成雷达图、柱状图、风险矩阵图
- 输出详细的对比分析报告

## 四、程序复杂度分析

### 4.1 时间复杂度

**整体时间复杂度**：O(n³)

**分模块分析**：

1. **AHP权重计算**（`AHPAnalyzer.calculate_weights()`）
   - 特征值分解：O(n³)，其中n为判断矩阵维度
   - 本程序中n=6，复杂度为O(216)
   - **实际耗时**：< 1ms

2. **数据标准化**（`normalize_data()`）
   - 遍历所有数据点：O(m×k)
   - m为稳定币数量（2个），k为指标数量（约20个）
   - **实际耗时**：< 1ms

3. **综合评分计算**（`calculate_detailed_scores()`）
   - 外层循环：O(m)，m=2
   - 内层循环：O(n×k)，n=6个一级指标，k≈3个二级指标
   - 总复杂度：O(m×n×k) = O(2×6×3) = O(36)
   - **实际耗时**：< 5ms

4. **风险矩阵计算**（`calculate_risk_matrix()`）
   - 向量点积：O(n)，n=5个风险类别
   - **实际耗时**：< 1ms

5. **可视化绘图**（`Visualizer`类）
   - 雷达图：O(n)，n=6个顶点
   - 柱状图：O(n)
   - 风险矩阵：O(m×k)，m=2种币，k=5个风险点
   - **实际耗时**：约200-300ms（主要是matplotlib绘图）

**总计运行时间**：约0.3-0.5秒（在标准PC上）

### 4.2 空间复杂度

**整体空间复杂度**：O(n²)

**分模块分析**：

1. **判断矩阵存储**：O(n²) = O(36)，约288字节
2. **权重向量**：O(n) = O(6)，48字节
3. **评价数据**：O(m×n×k) = O(2×6×3) = O(36)，约576字节
4. **结果数据**：O(m×n) = O(12)，约200字节
5. **图表缓存**：O(图片分辨率)，约3MB（3张图）

**峰值内存占用**：< 50MB（包括Python运行时和matplotlib）

### 4.3 性能优化建议

如果需要处理大规模数据（如评估100种稳定币），可考虑以下优化：

1. **并行计算**：使用`multiprocessing`并行计算各币种得分
2. **缓存机制**：对重复计算的权重向量进行缓存
3. **向量化运算**：充分利用NumPy的向量化特性
4. **延迟绘图**：仅在需要时才生成图表

**优化后预期性能**：
- 100种稳定币对比：< 2秒
- 1000种稳定币对比：< 15秒

## 七、常见问题与解决方案

### 问题1：中文字体显示为方框

**原因**：matplotlib默认不支持中文字体

**解决方案**：
```python
# 在代码中已设置，如仍有问题，尝试以下方法：

# macOS
plt.rcParams['font.sans-serif'] = ['Arial Unicode MS']

# Windows
plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei']

# Linux
plt.rcParams['font.sans-serif'] = ['WenQuanYi Micro Hei']
```

### 问题2：一致性检验不通过

**原因**：判断矩阵不满足一致性要求（CR≥0.1）

**解决方案**：
1. 检查判断矩阵是否满足互反性：aij = 1/aji
2. 减少极端判断（避免9:1的过大比例）
3. 保持判断的传递一致性：如A优于B，B优于C，则A应优于C

### 问题3：图表无法保存

**原因**：文件夹权限或路径不存在

**解决方案**：
```bash
# 确保目标文件夹存在
mkdir -p 第一个问题

# 检查文件夹权限
chmod 755 第一个问题
```

### 问题4：程序运行缓慢

**原因**：数据量过大或绘图过多

**解决方案**：
1. 关闭不必要的图表生成
2. 降低图表分辨率（dpi=150而非300）
3. 使用`plt.switch_backend('Agg')`关闭交互式显示

### 问题5：NumPy警告信息

**警告**：`RuntimeWarning: divide by zero`

**解决方案**：
```python
# 代码中已处理，如仍出现，添加：
np.seterr(divide='ignore', invalid='ignore')
```
